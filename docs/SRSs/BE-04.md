### PHẦN 1: TÀI LIỆU KỸ THUẬT (SRS) CHO TICKET [BE-04]

#### **Ticket: [BE-04] Create Resident (Thêm mới nhân khẩu)**

**1. Thông tin chung**

- **Assignee:** Backend Dev
- **Branch Name:** `feature/BE-04-create-resident`
- **Estimated Time:** 2h

**2. Mô tả (Description)**
API cho phép thêm mới một nhân khẩu (cư dân) vào hệ thống. Cần đảm bảo tính duy nhất của số CCCD.

**3. Yêu cầu kỹ thuật (Technical Specs)**

- **Endpoint:** `POST /api/residents`
- **Payload:**
  ```json
  {
    "fullName": "Nguyen Van A",
    "dob": "1990-01-01",
    "gender": "Nam",
    "identityCard": "001090000001",
    "hometown": "Ha Noi",
    "job": "Ky su",
    "apartmentId": "optional_id"
  }
  ```
- **Validation:** Sử dụng `zod` để validate các trường bắt buộc (`fullName`, `dob`, `identityCard`).

**4. Acceptance Criteria (Tiêu chí chấp nhận)**

- [x] **Uniqueness:** Kiểm tra số CCCD (`identityCard`) đã tồn tại trong DB chưa. Nếu trùng trả về lỗi `400 Bad Request`.
- [x] **Validation:** Các trường bắt buộc không được để trống. Ngày sinh phải đúng định dạng.
- [x] **Database:** Lưu thành công document vào collection `Residents`.
- [x] **Response:** Trả về thông tin nhân khẩu vừa tạo kèm status `201 Created`.
