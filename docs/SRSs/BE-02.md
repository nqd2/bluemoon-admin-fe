### PHẦN 1: TÀI LIỆU KỸ THUẬT (SRS) CHO TICKET [BE-02]

#### **Ticket: [BE-02] Admin Authentication (Login)**

**1. Thông tin chung**

- **Assignee:** Nguyễn Quý Đức (Fullstack Lead)
- **Branch Name:** `feature/BE-02-admin-login`
- **Estimated Time:** 2h

**2. Mô tả (Description)**
Xây dựng API đăng nhập dành cho Admin. Hệ thống sẽ xác thực email và mật khẩu, nếu hợp lệ sẽ trả về JWT Token để sử dụng cho các request tiếp theo.

**3. Yêu cầu kỹ thuật (Technical Specs)**

- **Endpoint:** `POST /api/auth/login`
- **Payload:**
  ```json
  {
    "email": "admin@example.com",
    "password": "password123"
  }
  ```
- **Dependencies:**
  - `jsonwebtoken`: Tạo và verify token.
  - `bcryptjs` (hoặc `bcrypt`): So sánh mật khẩu đã hash.
  - `zod`: Validate dữ liệu đầu vào.

**4. Acceptance Criteria (Tiêu chí chấp nhận)**

- [x] **Validation:** Kiểm tra `email` và `password` không được để trống. Email phải đúng định dạng.
- [x] **Authentication:**
  - Tìm user trong database theo email.
  - So sánh password gửi lên với password hash trong DB bằng `bcrypt`.
- [x] **Response Success:**
  - Nếu thông tin đúng, trả về status `200`.
  - Body chứa `token` (thời hạn 30 ngày) và thông tin user (trừ password).
- [x] **Response Error:**
  - Nếu sai email hoặc password, trả về status `401 Unauthorized`.
  - Message: "Invalid email or password".
