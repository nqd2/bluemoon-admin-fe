### PHẦN 1: TÀI LIỆU KỸ THUẬT (SRS) CHO TICKET [BE-11]

#### **Ticket: [BE-11] Get Fee List (Danh sách khoản thu)**

**1. Thông tin chung**

* **Assignee:** Backend Dev
* **Branch Name:** `feature/BE-11-get-fees`
* **Estimated Time:** 1h

**2. Mô tả (Description)**
API lấy danh sách các khoản thu trong hệ thống. Hỗ trợ lọc theo trạng thái (Active/Inactive) để hiển thị trên Dashboard hoặc trang quản lý.

**3. Yêu cầu kỹ thuật (Technical Specs)**

*   **Endpoint:** `GET /api/fees`
*   **Authentication:** Yêu cầu `Bearer Token`.
*   **Query Parameters:**
    *   `type`: Filter theo loại phí (Optional). VD: `?type=Service`.
    *   `limit`: Giới hạn số lượng (Optional). Default: 20.
*   **Response Structure:**
    ```json
    {
      "success": true,
      "count": 5,
      "data": [
        {
          "_id": "...",
          "title": "Phí dịch vụ T12",
          "type": "Service",
          "amount": 5000,
          "unit": "m2",
          "createdAt": "2025-12-01T00:00:00.000Z"
        }
      ]
    }
    ```

**4. Business Logic**

1.  Query DB collection `Fees`.
2.  Mặc định sắp xếp theo `createdAt` giảm dần (mới nhất lên đầu).
3.  Nếu có query param `type`, thêm điều kiện lọc `{ type: req.query.type }`.

**5. Acceptance Criteria**

* [ ] **Sorting:** Danh sách phải được sắp xếp theo thời gian tạo mới nhất.
* [ ] **Filtering:** API hoạt động đúng khi truyền tham số `type`.
* [ ] **Performance:** Đảm bảo response time < 200ms cho < 1000 bản ghi.
